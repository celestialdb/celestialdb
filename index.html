Simply `npx celestial openAPI.json ./` and get a comprehensive list of query and mutation hooks tailored to your backend.

Celestial also generates default selectors providing common operations over the data fetched from the backend.

It is like using a library written specifically for your backend APIs. You just have to make function calls.

Declaratively inject data into components: call the query hook and use the suitable selector.

Combine selectors to mould data into component suitable shape. Any change to source data will cascade down to derived data, ensuring a consistent UI experience.

    // call the hooks
    useGetColorsQuery()
    useGetTasksQuery()

    // inject data
    const todo = useSelector(selectTasksById(id))
    const allColors = useSelector(selectColors)

    // initialize mutation hook
    const [ addTask ] = usePostTasksMutation()

    // this will issue a POST on /tasks endpoint
    addTask({newTask: {text: "my new todo"}})

Updates are just as straight forward: just call the mutation hook with the usual arguments. All updates are optimistic by default.

Optimistic updates reflect in all components using the data, including derived data.

Celestial abstracts away all details of local state management, exposing intuitive hooks to make local state management as easy as making function calls.

    // initialize cache to manage local state
    useCacheInit("status",
        { All: 'all', Active: 3, Completed: 1,});

    // update cache
    const onStatusChange = (status) => {
        cacheUpdate("status", status)
    }
